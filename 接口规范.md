# Module 5: Settlement & Reputation 接口规范

## 模块概述

**模块名称**: Settlement & Reputation (结算与声誉模块)

**核心职责**:
1. 项目方 Accept / Challenge 漏洞
2. 超时自动接受
3. 奖励自动分配
4. AI Agent Calibration Score
5. 审计师 Reputation

**MVP简化**: 人工仲裁，不实现自动委员会

---

## 接口规范

### 基础信息

- **Base URL**: `http://localhost:3005/api/v1`
- **Content-Type**: `application/json`
- **认证方式**: 钱包签名 (后续版本添加)

---

## 1. Settlement API

### 1.1 接收漏洞 (对接模块4)

```
POST /settlement/vulnerabilities
```

**请求体**:
```json
{
    "vulId": "0x123...",          // 来自模块4的漏洞ID
    "taskId": "0x456...",         // 关联任务ID (模块1)
    "title": "Reentrancy in withdraw()",
    "description": "...",
    "bounty": "1000.00",          // USDC
    "submitter": "0x789...",      // 提交者钱包
    "projectAddress": "0xabc...", // 项目方地址
    "projectName": "MyProject"
}
```

**响应**:
```json
{
    "id": "uuid",
    "vulId": "0x123...",
    "status": "PENDING",
    "createdAt": "2024-01-01T00:00:00Z"
}
```

### 1.2 项目方接受漏洞

```
POST /settlement/vulnerabilities/:vulId/accept
```

**请求体**:
```json
{
    "projectAddress": "0xabc..."
}
```

### 1.3 项目方挑战漏洞

```
POST /settlement/vulnerabilities/:vulId/challenge
```

**请求体**:
```json
{
    "projectAddress": "0xabc...",
    "reason": "这不是有效漏洞",
    "bond": "20000000"             // 20 USDC (最小保证金)
}
```

### 1.4 接收AI提交 (对接模块2)

```
POST /settlement/vulnerabilities/:vulId/submissions
```

**请求体**:
```json
{
    "subVulId": "0xdef...",        // 来自模块4的子漏洞ID
    "submitter": "auditor-001",
    "walletAddress": "0x789...",
    "agentId": "agent-001",        // AI Agent ID
    "confidence": 95,             // AI置信度 0-100
    "qualityScore": 85,           // 质量评分
    "ranking": 1,                 // 来自模块4的排名
    "pocQuality": 8,
    "fixQuality": 9
}
```

### 1.5 结算奖励

```
POST /settlement/vulnerabilities/:vulId/settle
```

**请求体**:
```json
{
    "txHash": "0x...",            // 链上交易哈希
    "blockNumber": 12345678
}
```

**响应**:
```json
{
    "vulnerabilityId": "0x123...",
    "rewardDistribution": {
        "totalBounty": "1000.00",
        "distributions": [
            {
                "address": "0x789...",
                "ranking": 1,
                "baseReward": "700.00",
                "calibrationMult": 120,
                "finalReward": "840.00"
            },
            {
                "address": "0xabc...",
                "ranking": 2,
                "baseReward": "200.00",
                "calibrationMult": 100,
                "finalReward": "200.00"
            }
        ],
        "totalDistributed": "1040.00"
    }
}
```

### 1.6 获取漏洞列表

```
GET /settlement/vulnerabilities?status=PENDING&page=1&limit=20
```

### 1.7 获取漏洞详情

```
GET /settlement/vulnerabilities/:vulId
```

## 3. Calibration API

### 3.1 注册AI Agent

```
POST /calibration/agents
```

**请求体**:
```json
{
    "agentId": "agent-001",
    "name": "VulnHunter AI",
    "description": "Advanced vulnerability detection"
}
```

### 3.2 上报预测结果

```
POST /calibration/agents/:id/report
```

**请求体**:
```json
{
    "confidence": 95,             // 预测时的置信度
    "actuallyCorrect": true       // 最终是否正确
}
```

### 3.3 获取校准乘数

```
GET /calibration/agents/:id/multiplier
```

**响应**:
```json
{
    "agentId": "agent-001",
    "name": "VulnHunter AI",
    "calibration": 85.5,
    "multiplier": 110,
    "grade": "B (Good)"
}
```

### 3.4 计算校准奖励

```
GET /calibration/agents/:id/calibrate?baseReward=1000
```

**响应**:
```json
{
    "baseReward": 1000,
    "calibration": 85.5,
    "multiplier": 110,
    "calibratedReward": 1100,
    "grade": "B (Good)"
}
```

### 3.5 获取AI排行榜

```
GET /calibration/leaderboard?sortBy=calibration&limit=10
```

## 4. Webhook事件

### 4.1 可订阅事件

| 事件名称 | 触发时机 | 订阅模块 |
|---------|---------|---------|
| `VULNERABILITY_RECEIVED` | 收到新漏洞 | module1, module2 |
| `PROJECT_ACCEPTED` | 项目方接受 | module2, module4 |
| `PROJECT_CHALLENGED` | 项目方挑战 | module2, module3 |
| `SETTLEMENT_COMPLETED` | 结算完成 | module1, module2 |
| `ARBITRATION_STARTED` | 仲裁开始 | module2, module3 |
| `REPUTATION_UPDATED` | 声誉更新 | module2 |

### 4.2 Webhook Payload格式

```json
{
    "event": "SETTLEMENT_COMPLETED",
    "module": "module5",
    "timestamp": "2024-01-01T00:00:00Z",
    "data": {
        "vulId": "0x123...",
        "totalBounty": "1000.00",
        "distributions": [...]
    }
}
```

### 4.3 Webhook签名验证

```javascript
const crypto = require('crypto');

function verifySignature(payload, signature, secret) {
    const expected = crypto
        .createHmac('sha256', secret)
        .update(payload)
        .digest('hex');
    return signature === expected;
}
```

## 5. 智能合约接口

### 5.1 Settlement.sol

```solidity
// 接收漏洞
function receiveVulnerability(
    bytes32 vulId,
    bytes32 taskId,
    string calldata title,
    string calldata description,
    uint256 bounty,
    address submitter
) external;

// 接收AI提交
function receiveAISubmission(
    bytes32 vulId,
    bytes32 subVulId,
    address submitter,
    bytes32 agentId,
    uint256 confidence,
    uint256 qualityScore,
    uint256 ranking
) external;

// 项目方接受
function acceptVulnerability(bytes32 vulId) external;

// 项目方挑战
function challengeVulnerability(bytes32 vulId, string calldata reason) external payable;

function settleReward(bytes32 vulId, uint256[] calldata calibrationMultipliers) external;
```

```solidity

function registerAuditor(address _auditor, string calldata _name) external;

// 更新审计师声誉
function updateAuditorReputation(address _auditor, bool isValid, uint256 bounty) external;

function registerAIAgent(bytes32 _agentId, string calldata _name) external;

// 更新AI校准
function updateAIAgentCalibration(bytes32 _agentId, uint256 confidence, bool actuallyCorrect) external;

// 获取校准乘数
function getCalibrationMultiplier(bytes32 _agentId) external view returns (uint256);
```

## 7. 数据字段说明

### 7.1 漏洞状态

| 状态 | 说明 |
|-----|------|
| `PENDING` | 待处理，等待项目方响应 |
| `ACCEPTED` | 项目方已接受 |
| `CHALLENGED` | 项目方已挑战 |
| `ARBITRATING` | 仲裁中 |
| `VALID` | 仲裁裁决有效 |
| `INVALID` | 仲裁裁决无效 |
| `SETTLED` | 已结算 |

### 7.2 审计师等级

| 等级 | 声誉阈值 | 最大提交数 |
|-----|---------|-----------|
| Rookie | 0 | 5 |
| Bronze | 100 | 8 |
| Silver | 500 | 11 |
| Gold | 2000 | 14 |
| Elite | 5000 | 17 |

### 7.3 AI校准等级

| 校准度 | 乘数 | 等级 |
|-------|------|------|
| >= 90% | 1.2x | A (Excellent) |
| 80-89% | 1.1x | B (Good) |
| 70-79% | 1.0x | C (Average) |
| 60-69% | 0.9x | D (Below Average) |
| < 60% | 0.5x | F (Poor) |

## 10. 示例请求

### 10.1 完整结算流程

```bash
# 1. 模块4聚类完成后，发送漏洞到模块5
curl -X POST http://localhost:3005/api/v1/settlement/vulnerabilities \
  -H "Content-Type: application/json" \
  -d '{
    "vulId": "0xabc123",
    "taskId": "0xtask001",
    "title": "Integer Overflow",
    "description": "...",
    "bounty": "1000.00",
    "submitter": "0xuser001",
    "projectAddress": "0xproj001"
  }'

# 2. 模块2上报AI提交
curl -X POST http://localhost:3005/api/v1/settlement/vulnerabilities/0xabc123/submissions \
  -H "Content-Type: application/json" \
  -d '{
    "subVulId": "0xsub001",
    "submitter": "user001",
    "walletAddress": "0xuser001",
    "agentId": "agent-001",
    "confidence": 95,
    "qualityScore": 85,
    "ranking": 1
  }'

# 3. 项目方接受
curl -X POST http://localhost:3005/api/v1/settlement/vulnerabilities/0xabc123/accept \
  -H "Content-Type: application/json" \
  -d '{"projectAddress": "0xproj001"}'

# 4. 结算奖励
curl -X POST http://localhost:3005/api/v1/settlement/vulnerabilities/0xabc123/settle \
  -H "Content-Type: application/json" \
  -d '{"txHash": "0xtx123"}'

# 5. 更新AI校准
curl -X POST http://localhost:3005/api/v1/calibration/agents/agent-001/report \
  -H "Content-Type: application/json" \
  -d '{"confidence": 95, "actuallyCorrect": true}'

# 6. 更新审计师声誉
curl -X POST http://localhost:3005/api/v1/reputation/auditors/user001/update \
  -H "Content-Type: application/json" \
  -d '{"isValid": true, "bounty": "1000.00"}'
```
